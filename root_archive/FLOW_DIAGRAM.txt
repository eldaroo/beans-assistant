EXAMPLE FLOWS:
═══════════════════════════════════════════════════════════════════════════════

1️⃣  ANALYTICAL QUERY (READ_ANALYTICS)
────────────────────────────────────────────────────────────────────────────

Input: "¿cuántas pulseras tengo de cada tipo?"

Flow:
  User → Router (classify: READ_ANALYTICS)
       → Read Agent (SQL: SELECT ... FROM stock_current)
       → Final Answer

Output:
  "You have:
   • BC-BRACELET-BLACK: 37 units
   • BC-BRACELET-CLASSIC: 45 units
   • BC-BRACELET-GOLD: 24 units"


2️⃣  WRITE OPERATION (WRITE_OPERATION)
────────────────────────────────────────────────────────────────────────────

Input: "registrame una venta de 20 pulseras negras"

Flow:
  User → Router (classify: WRITE_OPERATION / REGISTER_SALE)
       → Resolver (resolve "pulseras negras" → product_id: 2)
       → Write Agent (execute: register_sale(...))
       → Final Answer

Output:
  "✓ Sale registered (ID: 6, Total: $28.00)
     Current revenue: $1100.00
     Current profit: $850.00"


3️⃣  MIXED OPERATION (WRITE + READ)
────────────────────────────────────────────────────────────────────────────

Input: "vendí 2 pulseras black, ¿cómo queda el stock?"

Flow:
  User → Router (classify: MIXED)
       → Resolver (resolve "pulseras black" → product_id: 2)
       → Write Agent (execute: register_sale(...))
       → Read Agent (SQL: SELECT stock_qty ... WHERE product_id = 2)
       → Final Answer

Output:
  "✓ Sale registered (ID: 7, Total: $4.00)
     Current revenue: $1104.00

   Current stock for BC-BRACELET-BLACK: 35 units"


4️⃣  AMBIGUOUS REQUEST
────────────────────────────────────────────────────────────────────────────

Input: "registrar algo"

Flow:
  User → Router (classify: AMBIGUOUS, missing_fields: [...])
       → Final Answer

Output:
  "I need more information. Missing: operation type, details
   For example:
   • 'Register a sale of 5 black bracelets' (operation)
   • 'Show me current stock' (analytics)"

STATE FLOW:
═══════════════════════════════════════════════════════════════════════════════

AgentState (TypedDict) flows through all nodes:

┌──────────────────────────────────────────────────────────────┐
│ AgentState                                                   │
├──────────────────────────────────────────────────────────────┤
│ messages: List[Dict]           # Message history            │
│ user_input: str                # Original input             │
│ intent: IntentType             # Classification             │
│ operation_type: OperationType  # Specific operation         │
│ confidence: float              # 0-1 confidence             │
│ missing_fields: List[str]      # Missing data               │
│ normalized_entities: Dict      # Resolved entities          │
│ sql_result: str                # Read agent result          │
│ operation_result: Dict         # Write agent result         │
│ final_answer: str              # Final response             │
│ error: str                     # Error if any               │
│ next_action: str               # Flow control               │
└──────────────────────────────────────────────────────────────┘

Each agent reads this state and returns updates.
LangGraph merges updates as the state flows through nodes.
