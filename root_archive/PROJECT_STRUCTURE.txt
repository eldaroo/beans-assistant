🧩 COMPONENT BREAKDOWN
═══════════════════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────────────────┐
│ 1️⃣  INTENT ROUTER AGENT (agents/router.py)                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Classifies user intent                                                    │
│ • Extracts entities from natural language                                   │
│ • Identifies missing fields                                                 │
│ • NO execution of tools                                                     │
│ • Uses structured output (Pydantic)                                         │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 2️⃣  READ/ANALYTICS AGENT (agents/read_agent.py)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│ • SQL SELECT queries only                                                   │
│ • Uses business views (stock_current, profit_summary, etc.)                │
│ • Respects strict business rules                                           │
│ • NO writes to database                                                     │
│ • Human-readable explanations                                               │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 3️⃣  WRITE/OPERATIONS AGENT (agents/write_agent.py)                          │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Executes business operations (Python functions)                           │
│ • register_sale, register_expense, register_product, add_stock             │
│ • Validates data before execution                                           │
│ • NO raw SQL generation                                                     │
│ • Summarizes business impact                                                │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ 4️⃣  NORMALIZATION/RESOLVER AGENT (agents/resolver.py)                       │
├─────────────────────────────────────────────────────────────────────────────┤
│ • Resolves human references to database entities                            │
│ • "pulseras negras" → product_id                                            │
│ • "ayer" → ISO date                                                         │
│ • "$30" → cents                                                             │
│ • Validates required fields                                                 │
│ • NO writes, NO final decisions                                             │
└─────────────────────────────────────────────────────────────────────────────┘

💾 STATE MANAGEMENT
═══════════════════════════════════════════════════════════════════════════════

AgentState (TypedDict):
  ├── messages: List[Dict]           # Conversation history
  ├── user_input: str                # Original user input
  ├── intent: IntentType             # Classification result
  ├── operation_type: OperationType  # Specific operation
  ├── confidence: float              # Classification confidence
  ├── missing_fields: List[str]      # Missing required data
  ├── normalized_entities: Dict      # Resolved entities
  ├── sql_result: str                # Read agent output
  ├── operation_result: Dict         # Write agent output
  ├── final_answer: str              # Final response
  ├── error: str                     # Error if any
  └── next_action: str               # Flow control

  3. python init_db.py
