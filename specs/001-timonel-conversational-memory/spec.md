# Timonel conversational memory and crash bound-fix

> Spec generated by `/create-spec` on 2026-05-15 from a 7-agent deliberation.
> Deliberation artifacts under `./.deliberation/`.

## Problem

Timonel (the LangGraph entry node added in PR #36) has no awareness of prior turns. Memory today is a class-level deque inside `backend/services/chat_service.py`, scoped to one FastAPI worker process, lost on every Nomad restart, and only reachable from the web widget surface. Every container restart drops every tenant's history at the same instant. Two visible failures rode in on a single screenshot from Dario:

1. Turn 1 said "¿qué productos están por agotarse?". Turn 2 said "agreguemos". Timonel had no record of turn 1, so the router could not collapse "agreguemos" against the prior topic. The user got a disambiguation question and a side-effect at the same time (the chat widget's `inferNavigation()` keyword-scanned the bot's reply for "gasto" and opened the Gastos tab while the agent had not yet picked an intent).
2. Turn 3 said "un prodcuo al inventario" (typo on "producto"). The graph raised somewhere downstream and the response collapsed to the chat widget's catch-all `ERROR_FALLBACK_COPY['llm_unavailable']` ("No pude procesar tu mensaje ahora"). The user cannot tell a typo from a model outage. Operators cannot either.

The deeper problem is one architectural gap viewed from two angles: the graph treats every turn as cold, and treats every node exception as terminal and indistinct. Memory durability and per-node fault containment are the same shape.

## Why now

Real users are typing into Timonel today. Container restarts have happened multiple times in the last week (the WhatsApp connector restarted on 2026-05-13 from a template re-render, the backend has restarted with deploys). Each restart wipes memory across the whole tenant base. The screenshot shows a single conversation crashing on a single typo with no recovery copy. The longer this ships as-is, the more the platform trains tenants to distrust their own input.

A separate Nomad fix (`Light-Brands/infrastructure#40`) is in flight to flip the backend `/health` from `degraded` to healthy by giving the backend a working internal route to the WhatsApp connector. That fix and this spec together are what brings beans-assistant back to a state Dario can confidently demo.

## In scope

- A persistent, per-tenant conversation buffer that survives container restarts and is read by Timonel and the router on every turn.
- A single `MemoryStore` Protocol with one shipped implementation (per-tenant SQLite, the project's existing tenant model). Postgres implementation behind an env switch is allowed but not required for v1.
- Per-node fault isolation in the LangGraph (a `safe_node` wrapper) so a downstream exception writes a typed `error_class` to state and never raises into the dispatcher.
- A typed `error_code` round-trip from backend to widget, with named Spanish copy per failure mode (typo or unknown product, missing field, network, llm-unavailable). Incident-id surfaced in copy and in logs.
- Replacement of `chat_widget.js` `inferNavigation()` keyword-scan with a `metadata.navigation` field set by the agent only when a tool actually executed.
- Multi-tenant isolation invariant enforced at the storage layer, not at the application layer. One round-trip integration test that proves a write under tenant A is invisible under tenant B.
- Retention policy: 30-day TTL on the buffer, nightly purge job, an `olvidá todo` user command that clears history within 60s, and an export endpoint for the tenant.
- Spanish onboarding consent line shown before first memory write.
- Regression tests that replay the screenshot's exact transcript verbatim and assert correct routing on turn 2 and a clarifying response (not the generic crash copy) on turn 3.

## Out of scope

- An episode store, lesson extractor, or semantic recall over old turns. These belong in a follow-up spec; QI Master flagged that a lesson loop inside the request path causes prompt drift and should be a weekly batch instead.
- Any change to the decomposer's regex gate (`should_decompose`) beyond reading the prior intent. The split-LLM path is unchanged.
- AIQ / MIQ / TIS scoring on chat turns. QI Master flagged this as over-reach for a customer surface.
- A central Postgres `chat_history` table as the only backend. Postgres is a supported `MEMORY_BACKEND` value in v1 but not the default; defaults stay aligned with the existing per-tenant SQLite model.
- A redesign of the chat widget's visible UI for memory. Kaze's call: zero pixels of new chrome. Memory is felt by behavior, not signaled.

## Acceptance criteria

- [ ] Sending "agreguemos" immediately after "¿qué productos están por agotarse?" classifies as `REGISTER_PRODUCT` or `ADD_STOCK`, never `REGISTER_EXPENSE`. Covered by `tests/eval/router_ambiguity.json` with the screenshot transcript verbatim.
- [ ] Sending "un prodcuo al inventario" returns a Spanish clarifying response with `metadata.error_code` set (one of `unknown_product`, `missing_field`, `ambiguous_input`); the generic `llm_unavailable` copy never fires for this input; the response includes `metadata.incident_id` so the operator can correlate the turn with the structured log line. Covered by `tests/unit/test_chat_service_crash.py`. (Naming the typo'd candidate as "producto" is desirable but requires a typo-tolerant resolver, which is out of scope for v1; a follow-up spec can add fuzzy product matching once the error envelope is in place.)
- [ ] After a Nomad alloc restart of `beans-assistant`, the next chat turn renders prior history in the contextualized prompt. Verified by the runbook at `specs/001-timonel-conversational-memory/runbooks/restart-survival.md`.
- [ ] An integration test writes 6 turns under phone A and asserts a read under phone B returns zero rows. Test fails if the per-tenant DB boundary is broken.
- [ ] Every graph node exception is wrapped in `safe_node` and produces a state delta with `{class, node, msg, incident_id}`. The dispatcher never raises into the API layer. Covered by one unit test per node.
- [ ] The chat widget's reply rendering uses `metadata.error_code` when present and the extended `ERROR_FALLBACK_COPY` table covers `unknown_product`, `missing_field`, `network`, `llm_unavailable`. Covered by a JS unit test or DOM snapshot.
- [ ] The chat widget no longer calls `inferNavigation()` on assistant text. Navigation events fire only when `metadata.navigation` is set by the agent. Regression test: Turn-2 of the screenshot transcript no longer flips the UI to Gastos.
- [ ] The Spanish command "olvidá todo" purges the per-tenant conversation buffer and returns a confirmation reply within 60 seconds. Covered by an integration test against the SQLite store.
- [ ] Memory pre-fetch latency is bounded at 200ms. If the read exceeds the budget, the graph proceeds with empty memory. Covered by a test that injects a slow store.
- [ ] An onboarding screen at `backend/templates/onboarding.html` shows the consent line "Voy a recordar lo que charlamos para entenderte mejor. Podés borrar todo cuando quieras escribiendo 'olvidá todo'." before the first memory write.

## Open questions

- **Q1.** Storage backend default. Amelia, Bob, and QI Master argue per-tenant SQLite (matches the existing tenant DB pattern). Winston, Kaze, Edut argue a central Postgres `chat_history` table (matches `USE_POSTGRES=true` in production). Resolution proposed in plan: Protocol with both implementations, SQLite default, Postgres opt-in via `MEMORY_BACKEND=postgres`. Needs Dario's call on the default for production rollout.
- **Q2.** Episode and lesson layer scope. QI Master argues episodes belong in v1, lessons are a weekly batch, conscience scoring does not belong here at all. Other panelists implicitly agreed by leaving it out. Resolution proposed: defer episode store to a follow-up spec. Confirm Dario is OK deferring.
- **Q3.** WhatsApp parity. Bob, Amelia, Edut, and QI Master all flagged that the Baileys connector path may not flow through `ChatService.chat_with_tenant`. Memory feature in v1 still covers WhatsApp because both surfaces should converge on the same `MemoryStore`, but task T-020 first verifies this. If verification surfaces a divergent path, the spec needs amendment.
